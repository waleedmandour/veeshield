# Contributing to VeeShield

Thank you for your interest in contributing to VeeShield! We appreciate your help in making this project better.

## 🚀 Getting Started

### Prerequisites

- Node.js 18.x or later
- npm (comes with Node.js)
- Git

### Development Setup

```bash
# Fork and clone the repository
git clone https://github.com/YOUR_USERNAME/veeshield.git
cd veeshield

# Install dependencies
npm install

# Start development server
npm run dev
```

## 📝 Development Guidelines

### Code Style

- Use TypeScript for all new files
- Follow the existing code style
- Use meaningful variable and function names
- Add comments for complex logic

### Commit Messages

We follow conventional commits:

- `feat:` - New features
- `fix:` - Bug fixes
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting)
- `refactor:` - Code refactoring
- `test:` - Adding tests
- `chore:` - Maintenance tasks

Example: `feat: add ransomware detection rule`

### Branch Naming

- `feature/feature-name` - New features
- `fix/bug-name` - Bug fixes
- `docs/documentation-name` - Documentation

## 🧪 Testing

Before submitting a PR, ensure:

1. The application builds successfully
   ```bash
   npm run build
   ```

2. No linting errors
   ```bash
   npm run lint
   ```

3. The application runs correctly
   ```bash
   npm run dev
   ```

## 📦 Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit a pull request

### PR Checklist

- [ ] Code compiles without errors
- [ ] Follows code style guidelines
- [ ] Documentation updated (if needed)
- [ ] Tests pass (if applicable)
- [ ] Commit messages are clear

## 🐛 Reporting Bugs

When reporting bugs, please include:

1. Steps to reproduce
2. Expected behavior
3. Actual behavior
4. Screenshots (if applicable)
5. Environment (OS, Node version, etc.)

## 💡 Feature Requests

We welcome feature requests! Please:

1. Check if the feature already exists
2. Describe the feature in detail
3. Explain the use case
4. Provide examples if possible
